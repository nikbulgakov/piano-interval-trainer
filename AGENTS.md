# AGENTS.md

## Назначение проекта

Piano Interval Trainer — простое браузерное приложение для тренировки восходящих гармонических интервалов с физической MIDI-клавиатурой.

Главные приоритеты: корректная музыкальная логика, надёжный MIDI-ввод, быстрый интерфейс и минимальная сложность.

## Контекст перед работой

Перед изменениями прочитай:

1. `docs/PRODUCT.md` — поведение продукта и критерии приёмки.
2. `docs/ARCHITECTURE.md` — архитектурные границы.
3. `docs/DECISIONS.md` — принятые решения и допущения.
4. `PLANS.md` — текущий спринт и ручная приёмка.

Не загружай все документы повторно, если для задачи достаточно одного предметного файла.

## Текущий статус

- Базовый MVP завершён и принят пользователем. Спринт 1 нового цикла — глобальные обозначения — готовится к ручной приёмке.
- Не начинай следующий спринт, пока пользователь явно не принял предыдущий этап.
- После завершения этапа дай пользователю короткий сценарий ручной проверки и дождись результата.
- Перед началом любого нового этапа сначала перечисли планируемые действия и выполняй их только после явного подтверждения пользователя.

## Технические ограничения

- Клиентское SPA: React, TypeScript, Vite.
- Без backend, аккаунтов, аналитики и сетевой передачи MIDI-данных.
- Без глобального state manager и UI-фреймворка.
- Web MIDI изолирован в адаптере; музыкальная и тренировочная логика не зависят от браузерных API.
- Таймеры рассчитываются от абсолютного времени, а не от количества срабатываний `setInterval`.
- Все пользовательские тексты — на русском; названия типов, функций и файлов — на английском.
- Новая production-зависимость требует конкретного обоснования.

## Правила изменений

- Не расширяй MVP функциями из раздела «Не входит в MVP» без запроса пользователя.
- При изменении поведения обнови `docs/PRODUCT.md`.
- При новом техническом или продуктовом компромиссе добавь запись в `docs/DECISIONS.md`.
- При изменении порядка работ обнови `PLANS.md`.
- Держи компоненты небольшими, а доменную логику — чистой и тестируемой.
- Не связывай расчёт интервала с русскими строками интерфейса: источником истины служит число полутонов.
- MIDI Note On с velocity `0` обрабатывай как Note Off.
- Удержанные клавиши от прошлого задания не должны засчитывать новое задание.

## Проверка

Обязательная автоматическая проверка спринта — `npm run lint`. Production-сборку проверяет Vercel Preview.

MIDI-ввод нельзя полностью проверить без физического устройства, поэтому после связанных изменений обязательно укажи ручную проверку из `PLANS.md`.

## Definition of Done

Задача завершена, когда:

- поведение соответствует `docs/PRODUCT.md`;
- lint проходит;
- Vercel Preview успешно собран;
- нет необоснованных зависимостей и несвязанных изменений;
- документация обновлена, если изменились поведение, архитектура или план;
- пользователю передан конкретный сценарий ручной приёмки.
