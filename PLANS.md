# План разработки

## Правило выполнения

- Реализуется только один спринт за раз.
- В конце спринта проходят автоматические проверки.
- Пользователь получает конкретный чек-лист для самостоятельного теста.
- Следующий спринт начинается только после явного подтверждения пользователя.
- Если при проверке меняется продуктовая логика, сначала обновляются `docs/PRODUCT.md` и `docs/DECISIONS.md`.

## Sprint 0 — Спецификация и контекст

Статус: завершён и принят пользователем.

Результат:

- продуктовая спецификация;
- архитектура;
- журнал решений;
- постоянные инструкции для Codex и Claude Code;
- поэтапный план.

Приёмка:

- подтвердить или исправить шесть допущений из `docs/PRODUCT.md`;
- подтвердить React + TypeScript + Vite;
- подтвердить, что код не начинается до согласования Sprint 0.

## Sprint 1 — Каркас, MIDI и экранная клавиатура

Статус: завершён и принят пользователем.

Цель: доказать главный технический риск — браузер стабильно видит физическую клавиатуру.

Входит:

- Vite + React + TypeScript;
- базовая страница без дизайна продукта;
- запрос MIDI-разрешения;
- список и выбор входных устройств;
- обработка подключения/отключения;
- экранная клавиатура MIDI 48–83;
- подсветка Note On/Off;
- автоматические тесты нормализации MIDI-сообщений;
- команды test, typecheck, lint и build.

Не входит: настройка и запуск тренировки.

Ручная приёмка:

1. Открыть приложение в Chrome или Edge на `localhost`.
2. Подключить MIDI-клавиатуру и дать разрешение.
3. Убедиться, что отображается имя устройства.
4. Нажать белые и чёрные клавиши в диапазоне — подсветка появляется без заметной задержки.
5. Удержать несколько клавиш — все подсвечены одновременно.
6. Отпустить клавиши — подсветка снята.
7. Отключить и снова подключить USB — статус обновился без перезагрузки.

Автоматическая проверка реализации:

- 8 unit-тестов проходят;
- TypeScript typecheck проходит;
- ESLint проходит;
- production build проходит;
- браузерная проверка подтвердила 36 клавиш и отсутствие console errors.

## Sprint 2 — Настройка и музыкальное ядро

Статус: завершён и принят пользователем.

Цель: собрать корректную конфигурацию и независимо проверить музыкальные правила.

Входит:

- выбор 12 исходных нот;
- выбор 12 интервалов;
- режим и длительность;
- период для timed-режима;
- валидация формы;
- модели pitch class и interval;
- допустимые MIDI-пары в диапазоне;
- проверка точного ответа;
- shuffle bag генератор;
- unit-тесты краёв диапазона и всех интервалов;
- preview следующего задания без запуска сессии.

Не входит: рабочий таймер сессии и подсчёт результатов.

Ручная приёмка:

1. Выбрать только До, m3 и M3.
2. Убедиться, что preview показывает только эти два сочетания.
3. Проверить ошибки формы при пустом выборе и неверных числах.
4. Переключить режимы и убедиться, что период виден только в режиме «На время».

Автоматическая проверка реализации:

- таблицы 12 нот и 12 интервалов покрыты unit-тестами;
- допустимые MIDI-пары и точный ответ из двух клавиш покрыты тестами;
- shuffle-bag проходит все выбранные сочетания до повторения;
- валидация диапазонов и режимов покрыта тестами;
- браузерная проверка подтвердила форму, preview и отсутствие console errors.

## Sprint 3 — Последовательная тренировка

Статус: завершён и принят пользователем.

Цель: получить первый полностью полезный end-to-end режим.

Входит:

- старт и досрочное завершение сессии;
- общий countdown;
- последовательная state machine;
- правильная/неправильная обратная связь;
- ожидание отпускания клавиш;
- счётчики;
- экран результата;
- unit-тесты переходов состояния и scoring.

Ручная приёмка:

1. Запустить короткую сессию только для `До — m3`.
2. Нажать неправильную пару — задание не сменилось, ошибка учтена.
3. Нажать C и E♭ с разницей 3 полутона — ответ засчитан.
4. Удерживать клавиши — следующее задание не засчитывается автоматически.
5. Отпустить клавиши — появляется следующее задание.
6. Дождаться конца времени — открывается корректная сводка.

Автоматическая проверка реализации:

- state machine последовательной сессии покрыта unit-тестами;
- ошибки считаются один раз на удерживаемую комбинацию;
- новое задание появляется только после полного отпускания клавиш;
- таймер основан на абсолютном deadline;
- экран настройки не содержит экранную клавиатуру;
- запуск заблокирован без MIDI и для timed-режима до Sprint 4;
- чистая браузерная загрузка проходит без console errors.

## Sprint 4 — Режим «На время»

Статус: завершён и принят пользователем.

Цель: ограничить время на задание без паузы после правильного ответа.

Входит:

- дедлайн для нерешённого задания;
- пропуск нерешённого задания;
- немедленная смена после правильного ответа с новым полным периодом;
- защита от удержанных клавиш на границе заданий;
- восстановление корректного времени после фоновой вкладки;
- тесты дедлайнов и пропусков.

Ручная приёмка:

1. Задать период 3 секунды и длительность 1 минуту.
2. Не отвечать — задания меняются каждые 3 секунды, пропуски растут.
3. Ответить раньше — следующее задание появляется сразу, а его таймер снова показывает 3 секунды.
4. Удерживать правильную пару после смены — новое задание не засчитывается до отпускания клавиш.
5. Свернуть вкладку и вернуть — общий таймер не растянулся.
6. Проверить итоговые правильные, ошибки, пропуски и точность.

Автоматическая проверка реализации:

- нерешённые задания меняются по абсолютным дедлайнам;
- правильный ответ сразу показывает следующее задание и запускает новый полный период;
- нерешённое задание при смене учитывается как пропуск;
- удержанные на границе клавиши требуют полного отпускания;
- после фоновой задержки пропущенные дедлайны обрабатываются последовательно;
- итоговая точность учитывает правильные ответы, ошибки и пропуски;
- браузерная проверка подтвердила валидацию периода, отсутствие горизонтального переполнения и ошибок консоли.

## Sprint 5 — Стабилизация локального MVP

Статус: завершён и принят пользователем.

Цель: убрать проблемы, найденные в реальной практике.

Входит:

- понятные состояния ошибок и восстановления MIDI;
- визуальная и клавиатурная доступность формы;
- адаптация под типичный экран ноутбука;
- финальная проверка Chrome и Edge;
- устранение дефектов ручной приёмки;
- обновление README и документации фактическими командами.

Ручная приёмка:

1. Пройти оба режима с несколькими нотами и интервалами.
2. Проверить отказ в разрешении MIDI и повторное подключение.
3. Проверить отключение устройства во время сессии.
4. Проверить читаемость задания и клавиатуры на небольшом экране ноутбука.
5. Подтвердить, что приложение достаточно удобно для регулярной практики.

Автоматическая проверка реализации:

- ошибки доступа к MIDI различают отказ в разрешении и общий сбой, для обоих доступно повторное подключение;
- успешное обновление или выбор MIDI-входа очищает устаревшее сообщение об ошибке;
- при отключении MIDI во время упражнения проверка ответов останавливается и доступен возврат к подключению;
- ошибки числовых полей связаны с полями через `aria-describedby`, состояния подключения объявляются через live regions;
- экранная клавиатура имеет отдельное текстовое состояние нажатых нот для screen reader;
- компактная компоновка практики включается для экранов высотой до 820 px;
- 42 unit-теста, TypeScript, ESLint и production build проходят;
- браузерная проверка на 1366×768 не выявила горизонтального переполнения или ошибок консоли.

## Новый цикл развития

### Спринт 1 — глобальные обозначения

Статус: завершён и принят пользователем.

Входит:

- отдельный экран настроек приложения;
- русские или латинские обозначения всех нот;
- полные названия или краткие символы всех интервалов;
- единое применение выбора во всех текущих экранах;
- сохранение настроек в `localStorage`;
- значения по умолчанию: русские ноты и полные интервалы.

Ручная приёмка на Vercel Preview:

1. При первом открытии убедиться, что используются русские ноты и полные названия интервалов.
2. Открыть настройки, выбрать латинские ноты и краткие интервалы, вернуться назад.
3. Проверить подписи в выборе задания, MIDI-индикаторе, практике и на экранной клавиатуре.
4. Перезагрузить страницу и убедиться, что выбор сохранился.
5. Проверить остальные две комбинации настроек и отсутствие смешанных подписей.
6. Подключить MIDI-клавиатуру и пройти короткую тренировку в обоих режимах.

Процесс приёмки:

- отдельная ветка и draft PR;
- обязательная автоматическая проверка — `npm run lint`;
- ручная проверка только на Vercel Preview;
- merge только после явного подтверждения пользователя.

### Спринт 2 — главный экран выбора тренировки

Статус: готов к ручной приёмке.

Входит:

- главный экран с карточками «Интервалы» и «Одиночные ноты»;
- переход к существующей настройке интервалов;
- недоступное состояние «Скоро» для ещё не реализованного поиска нот;
- доступ к глобальным настройкам с главного экрана;
- возврат из настройки интервалов без потери MIDI-состояния и параметров;
- понятные подписи возврата из настроек и результатов.

Ручная приёмка на Vercel Preview:

1. Открыть приложение и убедиться, что первым показывается выбор тренировки без запроса MIDI-доступа.
2. Проверить, что «Одиночные ноты» помечены «Скоро» и недоступны.
3. Открыть интервалы, подключить MIDI и изменить несколько параметров.
4. Вернуться на главный экран и снова открыть интервалы — подключение и параметры должны сохраниться.
5. Открыть глобальные настройки с главного экрана и из настройки интервалов; проверить корректный возврат в обоих случаях.
6. Пройти короткую тренировку и убедиться, что результат возвращает к настройке интервалов.

Процесс приёмки:

- отдельная ветка и draft PR;
- обязательная автоматическая проверка — `npm run lint`;
- ручная проверка только на Vercel Preview;
- merge только после явного подтверждения пользователя.

### Следующие согласованные спринты

1. Последовательная тренировка поиска одиночных нот.
2. Режим «На время» для поиска одиночных нот.

Каждый следующий спринт начинается только после отдельного описания, подтверждения и приёмки предыдущего.

## Идеи после согласованного цикла, только по отдельному запросу

- настраиваемый диапазон клавиатуры;
- нисходящие или мелодические интервалы;
- история и графики прогресса;
- звук и метроном;
- публикация на выбранном static host;
- PWA и offline-режим.
