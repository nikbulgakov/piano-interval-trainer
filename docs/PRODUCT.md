# Продуктовая спецификация

## 1. Цель

Пользователь тренируется быстро находить на физической MIDI-клавиатуре заданный восходящий гармонический интервал от указанной ноты.

Пример задания:

> До (C) — малая терция (m3)

Правильный ответ — одновременно удерживаемые До и Ми-бемоль, расположенные на расстоянии трёх полутонов вверх в доступном диапазоне клавиатуры.

## 2. Пользовательский сценарий

1. Пользователь открывает приложение в поддерживаемом браузере.
2. Нажимает «Подключить MIDI» и разрешает доступ.
3. При наличии нескольких входов выбирает нужное устройство.
4. Выбирает исходные ноты, интервалы, режим и длительность тренировки.
5. Для режима «На время» задаёт время на одно задание.
6. Запускает тренировку.
7. Видит задание, оставшееся время и экранную клавиатуру.
8. Играет интервалы; приложение подсвечивает физически нажатые клавиши и проверяет ответ.
9. По окончании времени получает краткий результат и может вернуться к настройкам.

## 3. Экран настройки

### 3.1. MIDI

- Кнопка явного запроса доступа к MIDI.
- Статусы: не подключено, запрос доступа, подключено, доступ запрещён, API не поддерживается, устройство отключено.
- Селектор MIDI-входа, если доступно больше одного устройства.
- Подключение и отключение устройства обновляются без перезагрузки страницы.
- После отказа в разрешении или ошибки подключения пользователь видит понятную причину и может повторить запрос.
- Используется только MIDI-вход; SysEx и MIDI-выход не нужны.

### 3.2. Исходные ноты

Пользователь выбирает один или несколько из 12 звуковысотных классов:

- До (C)
- До♯/Ре♭ (C♯/D♭)
- Ре (D)
- Ре♯/Ми♭ (D♯/E♭)
- Ми (E)
- Фа (F)
- Фа♯/Соль♭ (F♯/G♭)
- Соль (G)
- Соль♯/Ля♭ (G♯/A♭)
- Ля (A)
- Ля♯/Си♭ (A♯/B♭)
- Си (B)

Октавный номер в задании не показывается. Ответ можно сыграть от любого подходящего в тренировочном диапазоне экземпляра выбранной ноты.

### 3.3. Интервалы

MVP включает:

| Полутоны | Название | Обозначение |
|---:|---|---|
| 1 | малая секунда | m2 |
| 2 | большая секунда | M2 |
| 3 | малая терция | m3 |
| 4 | большая терция | M3 |
| 5 | чистая кварта | P4 |
| 6 | тритон | TT |
| 7 | чистая квинта | P5 |
| 8 | малая секста | m6 |
| 9 | большая секста | M6 |
| 10 | малая септима | m7 |
| 11 | большая септима | M7 |
| 12 | чистая октава | P8 |

Чистая прима исключена как не дающая полезного поиска второй клавиши.

### 3.4. Режимы

#### Последовательный

- Одно задание остаётся на экране до правильного ответа.
- Неправильный ответ даёт короткую визуальную обратную связь, но не меняет задание.
- После правильного ответа приложение ждёт отпускания всех клавиш и показывает следующее задание.
- Сессия завершается, когда истекает общая длительность, даже если текущее задание не решено.

#### На время

- Пользователь задаёт время на одно задание от 1 до 30 секунд.
- После правильного ответа следующее задание появляется сразу, не дожидаясь остатка периода.
- Для немедленно показанного задания начинается новый полный период.
- Если задание не решено до смены, оно учитывается как пропущенное.
- После смены задания удержанные ранее клавиши не проверяются, пока пользователь не отпустит все клавиши.

### 3.5. Длительность

- Допустимая продолжительность: от 1 до 60 минут.
- Интерфейс может предлагать пресеты и ручной ввод.
- Для расчёта окончания используется фактическое абсолютное время, чтобы фоновые задержки браузера не растягивали сессию.

### 3.6. Валидация перед стартом

Кнопка старта недоступна, если:

- MIDI не готов;
- не выбрана ни одна исходная нота;
- не выбран ни один интервал;
- длительность вне допустимого диапазона;
- в режиме «На время» время на задание вне допустимого диапазона.

## 4. Экран тренировки

Обязательные элементы:

- крупное название исходной ноты;
- полное название и сокращение интервала;
- оставшееся время сессии;
- простой счётчик результата;
- статус последнего ответа: нейтральный, правильно, ошибка;
- кнопка досрочного завершения;
- экранная клавиатура на три октавы в нижней части экрана.

Настройки во время активной сессии не редактируются.

Если MIDI-устройство отключается во время сессии, проверка ответов останавливается, отображается предупреждение и кнопка возврата к подключению. Общий таймер сессии при этом не приостанавливается.

## 5. Экранная клавиатура и проверка ответа

- Фиксированный внутренний диапазон MVP: MIDI-ноты 48–83, то есть 36 клавиш от C до B на протяжении трёх октав.
- Октавные номера в интерфейсе не показываются: разные MIDI-программы используют разные соглашения об их названиях.
- Note On подсвечивает клавишу; Note Off снимает подсветку.
- Note On с velocity `0` равен Note Off.
- Сообщения со всех MIDI-каналов обрабатываются одинаково.
- Педаль sustain и другие Control Change сообщения игнорируются.
- Ответ проверяется только по физически удерживаемым клавишам внутри диапазона.
- Правильный ответ — ровно две удерживаемые клавиши: нижняя имеет требуемый звуковысотный класс, а верхняя удалена на заданное число полутонов.
- Дополнительная удерживаемая клавиша делает текущую комбинацию неправильной.
- Порядок нажатия двух правильных клавиш не важен.

Пример: для `До (C) — m3` допустимы любые пары `[C, E♭]` с разницей ровно 3 MIDI-ноты, если обе входят в диапазон 48–83.

## 6. Выбор заданий

- Множество заданий — декартово произведение выбранных нот и интервалов.
- Используется перемешанный «мешок»: каждое сочетание появляется один раз до нового перемешивания.
- Одно и то же задание не повторяется подряд, если доступно больше одного сочетания.
- Если выбрано единственное сочетание, оно закономерно повторяется.
- В задании не показывается целевая верхняя нота, иначе упражнение теряет смысл.

## 7. Результат сессии

Минимальная сводка:

- длительность;
- число правильных ответов;
- число неправильных попыток;
- число пропущенных заданий для режима «На время»;
- точность: `правильные / (правильные + ошибки + пропуски)`.

История сессий не сохраняется в MVP.

## 8. Нефункциональные требования

- Вся работа выполняется локально в браузере.
- MIDI-сообщения не отправляются в сеть.
- Интерфейс не должен заметно задерживать подсветку клавиш.
- Основной целевой экран — ноутбук или настольный компьютер.
- Основные действия формы доступны с клавиатуры, а ошибки и динамические MIDI-состояния имеют текстовые подписи для вспомогательных технологий.
- Интерфейс остаётся понятным без музыкальной нотации на нотном стане.
- Ошибка MIDI или неподдерживаемый API не приводят к пустому экрану.

## 9. Не входит в MVP

- нисходящие и мелодические интервалы;
- прима;
- нотный стан и чтение нот;
- генерация звука;
- поддержка sustain при проверке;
- пользовательские диапазоны клавиатуры;
- статистика между сессиями, аккаунты и синхронизация;
- мобильная версия как основной сценарий;
- PWA/offline-установка;
- локализация на другие языки.

## 10. Подтверждённые продуктовые решения

Пользователь подтвердил перед Sprint 1:

1. Тренируются только восходящие гармонические интервалы.
2. Для ответа нужно одновременно удерживать две клавиши.
3. Диапазон ограничен MIDI 48–83 и не настраивается.
4. Чистая прима исключена.
5. В timed-режиме правильный ответ сразу показывает следующее задание и запускает для него новый полный период.
6. Используются русские названия с международным обозначением в скобках.
