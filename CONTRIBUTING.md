# Разработка Key Sense

## Рабочий процесс

1. Обновить `main`.
2. Реализовать одну связанную задачу без несогласованного расширения MVP.
3. Обновить тесты и документацию, если изменилось поведение.
4. Выполнить `npm run lint`.
5. Создать небольшой осмысленный коммит в `main`.
6. Провести ручную приёмку локально; если изменение затрагивает MIDI или тренировку, проверить с физической MIDI-клавиатурой.
7. Выполнить push в `main` только после успешных проверок и явного подтверждения пользователя.

## Команды проверки

```powershell
npm ci
npm run lint
```

## Имена и коммиты

- Коммиты: `feat: add session history`, `fix: clear held notes after reconnect`, `docs: update practice checklist`.
- Один коммит должен решать одну понятную задачу.

## Definition of Done

- поведение соответствует `docs/PRODUCT.md`;
- lint проходит локально;
- MIDI-сценарии проверены вручную, если изменение касается ввода;
- в diff нет секретов, артефактов сборки и несвязанных файлов.
